CSS (Cascading Style Sheets) Nedir?

CSS (Basamaklanmis Stil Katmanlari) web sayfasini sekillendirmek icin kullanilan bir kodlama turudur. CSS kullanarak web sayfasinin gorunumunu ve duzenini degistirebbiliriz. Bir webbb sitesinin gorunumunun masaustu bilgisayarlar, tabletler ve mobil cihazlar gibi farkli ekranlarda nasil degistigini de tanimlayabiliriz. CSS, C++ veya Javascript gibi bir proglamlama dili degildir.Bunun nedeni, CSS'in amacinin web tarayicilari icin bicimlendirmeye(HTML'e) stil vermek olmasidir. HTML yalnizca icerigi "Isaretleyen" bir dildir -yani, belgeye belirli bir gorsel ve yapisal bicimlendirme verir.


Hem HTML hem de CSS dogalari geregi bildirimseldir. Bu, yalnizca bir seyleri acikladigi ve herhangi bir islem yapmadigi anlamina gelir. Tarayicilar CSS'i goruntuleme icin yorumlarken, bunu dogrusal bir sekilde yapar. Bunun nedeni, CSS'in dosyayi yukaridan asagiya okuyan tarayiciya talimatlar vermesidir. Bir kural belirlendiginde ve daha sonra dizin asagiya dogru degistirildiginde, tarayici ekranda goruntulenecek nihai sonuc olarak en son degerlendirmeyi alir. CSS'deki C'nin ifade ettigi sey budur. CSS'in cascade(kaskad) etkisi, tarayiciyi, kuralin fiziksel olarak ne zaman gerceklestigine bagli olarak, kurallari art arda ve gecersiz kilacak sekilde goruntuleyecek sekilde ayarlar. Cascade, oge icin birden cok CSS bildirimi ve birden cok stil sayfasinin kombinasyonu oldugunda, belirli bir oge icin uygulanan stiller arasindaki cakismayi cozme anlamina gelir. Sonuc Olarak,CSS gorundugu kadar kolay degildir. Anlamadan kullanmaya calisirsaniz web gelistirmede zorluk yasarsiniz. bu nedenle CSS ogrenmek bbir probramlama dili ogrenmek kadar onemlidir.

CSS Syntax

Oncelikle CSS'e HTML ogesini nasil bulacagini soylememiz gerekir. BUNU selector denen bir ozellik ile yapabiliriz. CSS'de bir selector, HTML ogelerini etiket adi, sinif adi, kimligi ve cok daha fazlasina gore bulmak icin kullanilir.
Boylelikle tanimli selector e (h1) bagli olarak, CSS artik yeni kurallarin nereye uygulanacagini anlayabilir.
Bunlarin yani sira CSS kodlarimizi .css uzantili bir dosya icerisinde tutar ve bu dosyayi HTML sayfasi icerisine bbasit bir kodla cagirabiliriz. Ornegin CSS dosyamizin adi "Hello.css" olsun ve bu dosyayi Html sayfaya eklemek icin HTML icine yazmamiz gereken kod su sekildedir.
Bir CSS selector tanimlamanin bircok farkli yolu vardir. Asagida selector turlerinin bazi orneklerini gorebiliriz.

  • Class Selector: HTML ogelerini sinif ozelliklerine gore bulur
  • ID Selector: Ogeleri kimligine gore bulur
  • Element Selector: Ogeleri etiket adlarina gore bulur

CSS teknik olarak kolaydir. Anlasilmasi gereken sinirli sayida kural vardir. Bununla birlikte, CSS'i zorlastiran potansiyal kural kombinasyonudur. Gorsel gereksinimler daha karmasik hale geldikce, CSS kurallarinizin surdurulebbbilirligini kontrol altinda tutmak icin yaratici ancak yapilandirilmis cozumlere ihtiyac vardir.

Tasarim yonleri pazarlamaya ve UXX tasarimcisina birakilirken, front-end gelistiricilerinin tasarimin nasil calistigini anlamalari da iyidir. Tasarimcilarin tumu gelistirici degildir ve bazen tarayicilarin ve piksel rehberliginin daha ince gereksinimlerini anlamayanlarla karsilasiyoruz. Dijital tasarimcilarin stil sayfalarinin inceliklerini ve potansiyel karmasikligini ogrenmesi gerektigi kadar, tasarim dilinde de konusabbilmelisiniz.
